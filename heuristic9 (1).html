9. Help Users Recognize, Diagnose, and Recover from Errors

Short Summary

The ninth usability heuristic, Help Users Recognise, Diagnose, and Recover from Errors, means that when something goes wrong, the system should clearly explain what happened and how to fix it. Error messages should use plain, polite language—avoiding technical terms—and give specific guidance for what the user can do next. For example, instead of showing a confusing code like “Error 404,” a website could say, “Page not found – please check the URL or go back to the homepage.” This heuristic helps users stay calm, understand the problem, and quickly recover, making the experience smoother and less frustrating.
